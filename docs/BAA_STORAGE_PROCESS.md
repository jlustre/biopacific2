# BAA Storage Process

Signed BAAs must be stored securely and access must be restricted to authorized personnel only. Recommended options:

- Store signed BAAs in a private, access-controlled folder (e.g., Google Drive, OneDrive, or a secure internal server).
- Do not store actual signed BAAs in the public code repository.
- In the repository, maintain a registry (see `docs/BAA_VENDOR_REGISTRY.md`) with references to storage location and status.
- Access to the storage location should be limited to compliance, legal, and executive staff.
- Document access requests and changes to BAA files.

**Example Storage Reference:**

- Google Drive: /HIPAA/BAA/Signed_BAAs
- Internal Server: //secure/hipaa/baa/

Update this process as your organization’s needs change.
